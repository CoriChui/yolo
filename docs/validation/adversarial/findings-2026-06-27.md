## What the red-team tried hardest to break and could NOT (verified solid)

- `git merge ... -m "subject" -m "YOLO-Feature: <slug>"` lands a parseable trailer (the
  `--trailer`-on-merge fix). ✓
- Slug substring collisions in the done grep — `grep -qx` is exact full-line match, so `csv`
  can't match `csv-export`. ✓
- Annotated tags with slashes (`yolo/done/<slug>`) create + verify cleanly. ✓
- Three-dot diff, base detection + ref-guarding, worktree cleanup order. ✓
- Routing block byte-identical between `CLAUDE.md` and the template. ✓
- Unified `yolo: <act>` commit prefixes — no leftovers. ✓
- Every config key cited is defined and vice-versa; `agents.finish` defined + read. ✓

## Lesson

The most valuable validation here was adversarial, not confirmatory — a review asked "is
this good?" would have passed the broken `gh pr merge --sha` and `git fetch` commands,
because they *read* plausible. Only an agent actively trying to run them caught it. Pair
every change with a hostile re-review, and turn every confirmed git-level finding into a
layer-1 harness case so it can never regress silently.
