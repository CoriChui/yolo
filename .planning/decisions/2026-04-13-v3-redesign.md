# Decision: YOLO v3 Redesign

**Date:** 2026-04-13
**Status:** Implemented

## Context

YOLO had two competing systems (v1 and v2) that didn't share plumbing. v1 had a
full release→feature→task pipeline with 26-source intake, YAML state management
(state.yaml + release.yaml + feature.yaml), and multi-agent coordination. v2 was
a simpler loop for individual developers but had no external context awareness.

Research across the ecosystem (GSD 52K stars, Superpowers 150K stars, gstack 71K
stars, DORA 2025, Anthropic context engineering docs) showed that successful
frameworks pick one problem, solve it completely, and stay composable.

## Decision

Merge v1 and v2 into one loop with graduated ceremony. YOLO's differentiator is
**deterministic quality enforcement** (shell scripts) and **git-evidenced state**
(reconciliation), not lifecycle management.

## What Changed

**8 changes implemented:**

1. **One loop** — unified `loop.md` replaces v1 feature workflow + v2 loop.
   Single entry point: `/yolo:start`. Three ceremony levels (bare, --context, --research).

2. **Casual intake** — `--context <path>` flag replaces the 26-source intake system.
   Read a file or URL before planning. No manifests, no REQ-xxx IDs.

3. **Flat state** — `.planning/.focus` (one line) + feature .md files + git commits.
   Deleted state.yaml, release.yaml, feature.yaml, 7-status state machine.

4. **Composable** — shell scripts work standalone with no YOLO setup. `/yolo:init`
   is a lightweight directory + hook installer, not a state.yaml generator.

5. **Trimmed agents** — 615 → 301 lines across 5 agents. Rationalizations tables
   moved to `reference/agent-guidelines.md` (not loaded into agent context).

6. **Hooks** — PreToolUse hook blocks destructive git ops. PostToolUse hook advises
   on test file modifications. Replaces honor-based prompt constraints.

7. **Context isolation** — explicit rules in loop.md for minimal context passing.
   Each agent gets only its required inputs, not the full feature file.

8. **Sensors** — commit.sh and validate-plan.sh exit 0 for advisory warnings, exit 1
   only for structural errors. `--json` flag for programmatic output. Deleted-test-file
   detection added.

## What Was Deleted

- `/yolo:release`, `/yolo:feature`, `/yolo:intake` commands and workflows
- state.yaml, release.yaml, feature.yaml schemas
- 26-source intake taxonomy with versioning and conflict resolution
- v1 agents: plan.md, verify.md, feature-breakdown.md
- v2/ directory (merged into main)
- spec.md (replaced by loop.md)
- 7-status feature state machine and TOCTOU guards
- Crash recovery heuristics based on timestamps

## What Was Kept

- All shell scripts + 243 tests (the crown jewels)
- Git-evidenced reconciliation (reconcile.sh)
- TDD enforcement in the execute agent
- Evidence-based verification (Iron Laws in check agent)
- Debug and decide workflows (standalone)
- Worktree-based isolation

## Trade-offs

- **Lost:** Release-level coordination, multi-agent parallel execution, requirements
  traceability with REQ-xxx IDs. These are enterprise features most users don't need.
- **Gained:** Simplicity, composability, faster onboarding, smaller context footprint,
  deterministic enforcement via hooks instead of honor system.
